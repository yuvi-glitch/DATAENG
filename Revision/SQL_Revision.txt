/*## PATTERN 1 — Window Functions: RANK, DENSE_RANK, ROW_NUMBER

### What is it
Assigns a number to each row within a partition. The difference:
- ROW_NUMBER → always unique, no ties (1,2,3,4)
- RANK → ties get same rank, next rank skips (1,1,3)
- DENSE_RANK → ties get same rank, no skip (1,1,2)

### Sample Problem
```sql
-- Rank sellers by total revenue
SELECT
    seller_id,
    SUM(payment_value) AS total_revenue,
    RANK() OVER (ORDER BY SUM(payment_value) DESC) AS revenue_rank,
    DENSE_RANK() OVER (ORDER BY SUM(payment_value) DESC) AS dense_rank,
    ROW_NUMBER() OVER (ORDER BY SUM(payment_value) DESC) AS row_num
FROM fct_orders
GROUP BY seller_id;
```*/

/*### Problem 1
Rank customers by total amount spent. Show top 10 customers with their rank. 
If two customers spent the same amount, they should get the same rank and the next rank should skip.*/

select  customer_id , rank ()  over ( order by sum(payment_value) ASC)  as  rn from fct_orders
where rn <=10;

/*### Problem 2
For each product category, rank products by number of times they were ordered.
Return only the top 3 products per category (use ROW_NUMBER to break ties).*/

select product_category_english, row_number( order by count(order_id) asc )  from fct_order_items   limit 3;

select product_category_english, row_number( order by count(order_id) asc ) as rn  from fct_order_items   
qualify rn <=3 ;

/*## PATTERN 2 — Running Totals and Cumulative Sums
### What is it
Calculate a running/cumulative value using SUM() OVER with ORDER BY and ROWS clause.

### Sample Problem
```sql
-- Running total of revenue month by month
SELECT
    DATE_TRUNC('month', order_purchase_timestamp) AS order_month,
    SUM(payment_value) AS monthly_revenue,
    SUM(SUM(payment_value)) OVER (
        ORDER BY DATE_TRUNC('month', order_purchase_timestamp)
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS cumulative_revenue
FROM fct_orders
GROUP BY 1
ORDER BY 1;
```*/

/*### Problem 1
Show each seller's monthly revenue and their cumulative revenue from the start 
of the dataset up to that month. Order by seller and month.*/

select sum(payment_value) as mnthl_rev , sum(sum(payment_value)) over(partition by seller_id  rows between unbounded preeceding and current row  ) from fct_orders

/*### Problem 2
For each state, show the running total of orders placed, ordered by 
order_purchase_timestamp. Also show what percentage of total orders 
each state has accumulated so far.*/
with cte as (select count(order_id) over( partition by seller_state  order by order_purchase_timestamp ) as monthly, count(order_id) over(partition by state) as total  from fct_orders) select  (monthly/total) * 100 as percent

/*## PATTERN 3 — Self Joins: Pairs and Hierarchies
### What is it
Join a table to itself to compare rows within the same table. Used for finding 
pairs, finding managers, or comparing consecutive rows.

### Sample Problem
```sql
-- Find pairs of orders from the same customer in the same state
-- placed within 7 days of each other
SELECT
    a.order_id AS order_1,
    b.order_id AS order_2,
    a.customer_id,
    a.order_purchase_timestamp AS date_1,
    b.order_purchase_timestamp AS date_2
FROM fct_orders a
JOIN fct_orders b
    ON a.customer_id = b.customer_id
    AND a.order_id < b.order_id
    AND DATEDIFF('day', a.order_purchase_timestamp, b.order_purchase_timestamp) <= 7;
```*/

/*### Problem 1
Find customers who placed more than one order and show the days between 
their first and second order. Use a self join on customer_id.*/

select a.customer_id , datediff ('day', a.order_purchase_timestamp, b.order_purchase_timestamp) 
from fct_rders a join fct_orders b on a.customer_id = b.customer_id  where count(order_id) > 1

/*### Problem 2
Find all pairs of sellers who operate in the same city. 
Return seller_id_1, seller_id_2, and city. Avoid duplicate pairs 
(A,B and B,A should appear only once).*/
select distinct  a.seller_id as seller_id_1  , b.seller_id as seller_id_2 from fct_order_items a join fct_order_items b on a.city=b.city
and a.seller_id < b.seller_id


/*## PATTERN 4 — Date Functions: DATEDIFF, DATE_TRUNC, Date Ranges
### What is it
Manipulate and compare dates. Key functions in Snowflake:
- DATE_TRUNC('month', col) → truncate to month start
- DATEDIFF('day', start, end) → difference in days
- DATEADD('day', n, col) → add n days
- EXTRACT(month FROM col) → extract part

### Sample Problem
```sql
-- Monthly order count and average delivery time
SELECT
    DATE_TRUNC('month', order_purchase_timestamp) AS order_month,
    COUNT(order_id) AS total_orders,
    AVG(DATEDIFF('day', order_purchase_timestamp, order_delivered_customer_dt)) AS avg_delivery_days
FROM fct_orders
WHERE order_delivered_customer_dt IS NOT NULL
GROUP BY 1
ORDER BY 1;*/

/*### Problem 1
Find all orders placed in Q4 2017 (October to December). 
Show month, total orders, and total revenue. Use DATE_TRUNC for grouping.*/

select month(order_purchase_timestamp), count(order_id),sum(revenue)  from fct_orders 
where month(order_purchase_timestamp) between 10 AND 12  and year(order_purchase_timestamp) = 2017
group by date_trunc('month',order_purchase_timestamp)

/*### Problem 2
Calculate how many days before or after the estimated delivery date 
each order actually arrived. Negative = early, Positive = late. 
Show distribution by state — average early/late days per state.*/

select datediff('day', estimated_delivery,delivery_day) as day_cnt,state,  from fct_orders 

## PATTERN 5 — String Functions: SUBSTRING, REPLACE, CONCAT

### What is it
Manipulate string columns. Key functions:
- SUBSTRING(col, start, length)
- REPLACE(col, 'old', 'new')
- CONCAT(col1, ' ', col2) or col1 || ' ' || col2
- UPPER(), LOWER(), TRIM()
- LEFT(col, n), RIGHT(col, n)
- LENGTH(col)

### Sample Problem
```sql
-- Create a display label combining city and state
SELECT
    customer_id,
    CONCAT(UPPER(city), ' - ', UPPER(state)) AS location_label,
    LEFT(customer_unique_id, 8) AS short_customer_id
FROM dim_customers;
```

### Problem 1
Product category names in the dataset have underscores (e.g. "bed_bath_table").
Replace all underscores with spaces and convert to uppercase for a clean display label.
Show product_id and the cleaned category name.

### Problem 2
Create a seller identifier by concatenating seller_id (first 6 chars) 
with their city (uppercase) separated by a dash.
Example output: "abc123-SAO PAULO". Filter only sellers with total_revenue > 10000.

## PATTERN 6 — Pivoting: Rows to Columns

### What is it
Convert row values into columns. In Snowflake use CASE WHEN inside SUM/COUNT 
to pivot manually (or use PIVOT operator).

### Sample Problem
```sql
-- Count orders by status for each state (pivot order_status)
SELECT
    state,
    COUNT(CASE WHEN order_status = 'delivered' THEN 1 END) AS delivered,
    COUNT(CASE WHEN order_status = 'shipped' THEN 1 END) AS shipped,
    COUNT(CASE WHEN order_status = 'canceled' THEN 1 END) AS canceled,
    COUNT(CASE WHEN order_status = 'processing' THEN 1 END) AS processing
FROM fct_orders
GROUP BY state
ORDER BY state;
```

### Problem 1
For each product category, show total revenue broken down by payment type 
(credit_card, boleto, voucher, debit_card) as separate columns.

### Problem 2
Show for each seller: total orders in 2017 and total orders in 2018 
as separate columns. Also add a column for the year over year difference.

## PATTERN 7 — Finding and Removing Duplicates

### What is it
Identify duplicate rows and keep only one. Use ROW_NUMBER() OVER 
(PARTITION BY key ORDER BY tiebreaker) = 1 to deduplicate.

### Sample Problem
```sql
-- Find duplicate orders (same customer, same purchase date)
SELECT
    customer_id,
    DATE(order_purchase_timestamp) AS order_date,
    COUNT(*) AS duplicate_count
FROM fct_orders
GROUP BY 1, 2
HAVING COUNT(*) > 1;

-- Remove duplicates, keep latest order_id
SELECT * FROM (
    SELECT *,
        ROW_NUMBER() OVER (
            PARTITION BY customer_id, DATE(order_purchase_timestamp)
            ORDER BY order_id DESC
        ) AS rn
    FROM fct_orders
)
WHERE rn = 1;
```

### Problem 1
In the raw customers table, customer_unique_id can appear multiple times 
with different customer_ids. Find all customer_unique_ids that appear 
more than once and count how many duplicates exist.

### Problem 2
From dim_products, some products may have the same category and same price.
Identify these duplicate product entries and return a deduplicated list 
keeping the first product_id alphabetically for each category-price combination.

## PATTERN 8 — NULL Handling: COALESCE, NULLIF, IFNULL

### What is it
- COALESCE(a, b, c) → returns first non-null value
- NULLIF(a, b) → returns NULL if a = b, else returns a (useful to avoid divide by zero)
- IFNULL(a, b) → returns b if a is null (Snowflake supports this)
- IS NULL / IS NOT NULL in WHERE clause

### Sample Problem
```sql
-- Handle null review scores and null delivery dates
SELECT
    order_id,
    COALESCE(avg_review_score, 0) AS review_score,
    COALESCE(delivery_days, -1) AS delivery_days,
    NULLIF(payment_value, 0) AS payment_value_clean,
    CASE WHEN order_delivered_customer_dt IS NULL 
         THEN 'Not Delivered' 
         ELSE 'Delivered' END AS delivery_status
FROM fct_orders;
```

### Problem 1
Some orders have NULL delivery_days because they were never delivered.
Show total orders, orders with delivery data, and orders without delivery data 
per state. Use COALESCE to replace NULL delivery_days with 0 for the average.

### Problem 2
Calculate average review score per seller. 
Some orders have no review — treat missing review scores as 3 (neutral).
Use COALESCE. Also flag sellers where ALL their orders have null reviews.

## PATTERN 9 — Set Operations: UNION, INTERSECT, EXCEPT

### What is it
- UNION → combine two result sets, remove duplicates
- UNION ALL → combine two result sets, keep duplicates (faster)
- INTERSECT → rows that exist in BOTH result sets
- EXCEPT → rows in first set that are NOT in second set

### Sample Problem
```sql
-- Find states that have both high revenue sellers AND high review scores
-- States with avg revenue > 50000
SELECT state FROM dim_sellers WHERE total_revenue > 50000
INTERSECT
-- States with avg review score > 4
SELECT state FROM fct_orders WHERE avg_review_score > 4;
```

### Problem 1
Using UNION ALL, create a combined list of all cities from both 
dim_customers and dim_sellers. Then count how many times each city 
appears across both tables (shows where customers and sellers overlap geographically).

### Problem 2
Find product categories that had orders in 2017 but did NOT have orders in 2018.
Use EXCEPT. This shows categories that dropped off completely year over year.

## PATTERN 10 — Correlated Subqueries

### What is it
A subquery that references a column from the outer query. 
Runs once per row of the outer query. Slower but sometimes necessary.

### Sample Problem
```sql
-- Find orders where payment_value is above that customer's average
SELECT
    order_id,
    customer_id,
    payment_value
FROM fct_orders f1
WHERE payment_value > (
    SELECT AVG(payment_value)
    FROM fct_orders f2
    WHERE f2.customer_id = f1.customer_id  -- correlated reference
);
```

### Problem 1
Find all sellers whose total_revenue is above the average total_revenue 
of sellers in their same state. Use a correlated subquery referencing 
the outer seller's state.

### Problem 2
For each order, find whether the delivery_days is more than 2x the 
average delivery_days for that state. Return order_id, state, delivery_days, 
and state average. Use a correlated subquery.

## PATTERN 11 — CASE WHEN for Categorisation

### What is it
Create conditional categories inline in SQL. Used for bucketing, 
labelling, and conditional aggregation.

### Sample Problem
```sql
-- Categorise orders by delivery speed and payment size
SELECT
    order_id,
    CASE
        WHEN delivery_days <= 3 THEN 'Fast'
        WHEN delivery_days <= 7 THEN 'Normal'
        WHEN delivery_days <= 14 THEN 'Slow'
        ELSE 'Very Slow'
    END AS delivery_category,
    CASE
        WHEN payment_value < 50 THEN 'Low'
        WHEN payment_value < 200 THEN 'Medium'
        ELSE 'High'
    END AS payment_bucket
FROM fct_orders;
```

### Problem 1
Categorise sellers into tiers based on total_revenue:
- Platinum: > 100000
- Gold: 50000 to 100000
- Silver: 10000 to 50000
- Bronze: below 10000
Show count of sellers in each tier.

### Problem 2
For each state, calculate the percentage of orders that are 
'On Time' vs 'Delayed' vs 'Unknown' (null delivery date).
Use CASE WHEN inside COUNT and divide by total orders per state.

## PATTERN 12 — Percentage and Ratio Calculations

### What is it
Calculate percentages using window functions or subqueries. 
Key pattern: value / SUM(value) OVER () * 100

### Sample Problem
```sql
-- Each state's share of total revenue
SELECT
    state,
    SUM(payment_value) AS state_revenue,
    ROUND(
        SUM(payment_value) * 100.0 / SUM(SUM(payment_value)) OVER (),
        2
    ) AS revenue_share_pct
FROM fct_orders
GROUP BY state
ORDER BY state_revenue DESC;
```

### Problem 1
For each product category, calculate:
- Total revenue for that category
- Category's percentage of overall total revenue
- Category's percentage within its state
Order by revenue descending.

### Problem 2
Calculate the on-time delivery rate per seller (orders delivered on time / total orders).
Also show how each seller's on-time rate compares to the overall average on-time rate.
Flag sellers who are below average as 'Underperforming'.

## PATTERN 13 — Top N Per Group

### What is it
Get top N rows within each group. Classic pattern using ROW_NUMBER() 
with PARTITION BY and then filtering WHERE rn <= N.

### Sample Problem
```sql
-- Top 3 sellers by revenue in each state
SELECT * FROM (
    SELECT
        state,
        seller_id,
        total_revenue,
        ROW_NUMBER() OVER (
            PARTITION BY state
            ORDER BY total_revenue DESC
        ) AS rn
    FROM dim_sellers
)
WHERE rn <= 3;
```

### Problem 1
Find the top 5 most ordered product categories per state.
Show state, category, order count, and rank within state.

### Problem 2
For each month in the dataset, find the top 3 customers by total spend.
Show month, customer_id, total_spend, and their rank for that month.

## PATTERN 14 — Gaps and Islands

### What is it
Gaps = missing entries in a sequence. Islands = consecutive unbroken sequences.
Classic technique: ROW_NUMBER() subtracted from date creates a group identifier
for consecutive dates.

### Sample Problem
```sql
-- Find consecutive months where a seller had orders (islands)
-- and gaps where they had no orders
WITH seller_months AS (
    SELECT
        seller_id,
        DATE_TRUNC('month', order_purchase_timestamp) AS order_month
    FROM fct_orders
    GROUP BY 1, 2
),
ranked AS (
    SELECT
        seller_id,
        order_month,
        ROW_NUMBER() OVER (PARTITION BY seller_id ORDER BY order_month) AS rn,
        DATEADD('month', 
            -ROW_NUMBER() OVER (PARTITION BY seller_id ORDER BY order_month),
            order_month
        ) AS grp
    FROM seller_months
)
SELECT
    seller_id,
    MIN(order_month) AS streak_start,
    MAX(order_month) AS streak_end,
    COUNT(*) AS consecutive_months
FROM ranked
GROUP BY seller_id, grp
ORDER BY seller_id, streak_start;
```

### Problem 1
Find sellers who had a gap of more than 2 consecutive months with no orders.
Show seller_id, the month they went inactive, and the month they returned.

### Problem 2
Find customers who placed orders in 3 or more consecutive months.
Show customer_id, streak start month, streak end month, and streak length.

## PATTERN 15 — NOT EXISTS and Anti Join

### What is it
Find rows in table A that have NO matching row in table B.
Three equivalent approaches — know all three:
- LEFT JOIN + IS NULL (most common)
- NOT EXISTS (safest with NULLs)
- NOT IN (dangerous with NULLs — avoid)

### Sample Problem
```sql
-- Customers who never left a review (LEFT JOIN + IS NULL)
SELECT c.customer_unique_id, c.city, c.state
FROM dim_customers c
LEFT JOIN fct_orders f ON c.customer_unique_id = f.customer_id
WHERE f.avg_review_score IS NULL;

-- Same using NOT EXISTS (safer)
SELECT customer_unique_id
FROM dim_customers c
WHERE NOT EXISTS (
    SELECT 1 FROM fct_orders f
    WHERE f.customer_id = c.customer_unique_id
    AND f.avg_review_score IS NOT NULL
);
```

### Problem 1
Find sellers who have never had a delayed order (is_on_time = 'Delayed' never appears).
Use NOT EXISTS. These are your most reliable sellers.

### Problem 2
Find product categories that exist in dim_products but have NEVER appeared 
in any order in fct_orders. Use LEFT JOIN + IS NULL pattern.
These are zero-sales categories.

## PATTERN 16 — Double NOT EXISTS (Relational Division)

### What is it
Find rows that satisfy ALL conditions in another table.
Classic use: "customers who ordered from ALL product categories"
or "sellers who sold in every state".

### Sample Problem
```sql
-- Find customers who have used ALL payment types
-- (ordered at least once with every payment type that exists)
SELECT DISTINCT customer_id
FROM fct_orders f1
WHERE NOT EXISTS (
    SELECT DISTINCT payment_type FROM fct_orders  -- all payment types
    EXCEPT
    SELECT payment_type FROM fct_orders f2
    WHERE f2.customer_id = f1.customer_id          -- this customer's payment types
);
```

### Problem 1
Find sellers who have received orders from ALL Brazilian states.
Use double NOT EXISTS or EXCEPT approach.

### Problem 2
Find customers who have ordered from at least 5 different product categories.
Then from that group, find who has ALSO used at least 3 different payment types.
Chain your conditions.

## PATTERN 17 — LAG and LEAD (Previous and Next Row)

### What is it
Access values from previous row (LAG) or next row (LEAD) 
within a partition without a self join.

### Sample Problem
```sql
-- Month over month revenue change
WITH monthly AS (
    SELECT
        DATE_TRUNC('month', order_purchase_timestamp) AS order_month,
        SUM(payment_value) AS monthly_revenue
    FROM fct_orders
    GROUP BY 1
)
SELECT
    order_month,
    monthly_revenue,
    LAG(monthly_revenue) OVER (ORDER BY order_month) AS prev_month_revenue,
    monthly_revenue - LAG(monthly_revenue) OVER (ORDER BY order_month) AS mom_change,
    ROUND(
        (monthly_revenue - LAG(monthly_revenue) OVER (ORDER BY order_month)) 
        * 100.0 / LAG(monthly_revenue) OVER (ORDER BY order_month),
        2
    ) AS mom_pct_change
FROM monthly;
```

### Problem 1
For each seller, show their revenue for each month, the previous month's revenue,
and flag months where revenue dropped more than 20% compared to the previous month.

### Problem 2
For each customer who placed multiple orders, show each order with 
the number of days since their previous order (use LAG on order_purchase_timestamp).
Find the customer with the longest gap between two consecutive orders.

## PATTERN 18 — EXISTS (Semi Join)

### What is it
Returns rows from table A where at least one match exists in table B.
More efficient than IN for large subqueries. Stops at first match found.

### Sample Problem
```sql
-- Find customers who placed at least one high value order (> 500)
SELECT DISTINCT customer_id, city, state
FROM dim_customers c
WHERE EXISTS (
    SELECT 1
    FROM fct_orders f
    WHERE f.customer_id = c.customer_unique_id
    AND f.payment_value > 500
);
```

### Problem 1
Find all states that have at least one seller with total_revenue above 50000.
Use EXISTS. Show the state name and count of such high revenue sellers.

### Problem 2
Find product categories where at least one product has been reviewed 
with a score of 5. Use EXISTS linking dim_products to fct_orders via 
int_order_items_enriched.

## Quick Reference — When to Use Which Pattern

| Scenario | Pattern to Use |
|---|---|
| Rank within group | ROW_NUMBER / RANK / DENSE_RANK |
| Running total | SUM() OVER (ORDER BY) |
| Compare to previous row | LAG / LEAD |
| Top N per group | ROW_NUMBER + WHERE rn <= N |
| Rows with no match | LEFT JOIN + IS NULL or NOT EXISTS |
| Rows satisfying ALL conditions | Double NOT EXISTS |
| Has at least one match | EXISTS |
| Consecutive sequences | Gaps and Islands (ROW_NUMBER - date trick) |
| Conditional aggregation | CASE WHEN inside SUM/COUNT |
| Rows above group average | Correlated subquery or window avg |
| Remove duplicates | ROW_NUMBER + WHERE rn = 1 |
| Month over month change | LAG window function |
| Share of total | value / SUM() OVER () |
| Combine result sets | UNION / INTERSECT / EXCEPT |

## Interview Tips

1. Always start your answer with "I would use a CTE to break this down"
2. For any ranking question — clarify "should ties be handled?" → shows maturity
3. For NOT EXISTS vs NOT IN → always say "I prefer NOT EXISTS because NOT IN 
   behaves unexpectedly when the subquery contains NULLs"
4. For window functions → always mention PARTITION BY and ORDER BY separately
5. For gaps and islands → draw the intermediate table on paper first, 
   then write the query — shows structured thinking
